# AI Account Switcher v0.5.11

This release hardens Claude session priming on macOS by avoiding the Claude Code agent runtime for
the one-token background request.

## What changed

- Claude **Prime ngay** and scheduled/wakeup primes now call the Anthropic Messages endpoint
  directly with the selected profile's OAuth token.
- Prime no longer starts `claude -p hi` for the normal send path, avoiding Apple Music / Media
  Library permission prompts caused by Claude Code sandbox/tool preflights.
- The existing auto-prime scheduler is unchanged: pmset wake, LaunchDaemon headless mode, retry
  deadlines, hold/defer handling, and reset confirmation still use the same runtime flow.
